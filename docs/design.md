# 设计文档

## 1. 项目概述

### 1.1 项目名称

一园入画·掌上云游

### 1.2 项目定位

本项目是一个基于 Web 的园林 3D 导览应用，使用园林总平面图作为总览入口，结合单体建筑 GLB 模型、建筑解说文本和音频旁白，构建“先览全园，再入单景”的沉浸式导览体验。

### 1.3 设计目标

- 让用户在移动端和桌面端都能快速理解园林空间关系。
- 让建筑浏览从“文件级展示”升级为“带叙事的导览体验”。
- 用统一的东方纸墨视觉语言承接地图、模型、文案和转场。
- 控制交互复杂度，让首次进入的用户无需学习成本即可完成浏览。

## 2. 用户与场景

### 2.1 目标用户

- 对古典园林、建筑文化、文旅导览感兴趣的普通游客。
- 希望通过数字化方式了解园林空间布局的学生与研究者。
- 在展陈、课堂、宣传场景中需要快速演示园林建筑亮点的讲解人员。

### 2.2 核心使用场景

- 在总览地图中浏览建筑分布，快速理解园林结构。
- 从地图或目录进入某个建筑，查看对应的 3D 模型。
- 在浏览模型的同时阅读图文解说，或直接收听音频讲解。
- 在不同建筑之间快速切换，保持导览节奏与叙事连贯性。

## 3. 信息架构

### 3.1 页面结构

项目当前只有一个主页面，但包含两种核心舞台状态：

- 总览态：展示园林平面图、建筑落点、预览卡片和建筑目录。
- 单体态：展示单个建筑的 3D 模型、建筑摘要、解说面板和目录入口。

### 3.2 导航方式

- 主导航方式为“状态切换”，不是多页面跳转。
- 用户在总览态点击建筑标签或目录项后，进入对应建筑的单体态。
- 用户在单体态点击左下角返回按钮后，回到总览态。
- 建筑之间的切换通过统一的墨迹转场完成。

### 3.3 内容对象

每个建筑对象包含以下内容：

- 基础标识：`slug`、`label`
- 展示文案：`summary`、`verse`
- 解说文案：`interpretation`
- 地图信息：`mapPosition`、`mapSize`
- 总览态文案：`overviewTag`、`overviewCopy`、`overviewHint`
- 资源文件：`{slug}.glb`、可选的 `public/audio/{slug}.mp3`

## 4. 核心交互设计

### 4.1 总览态

总览态是用户进入应用后的第一视图，承担“认知园林格局”和“选择建筑入口”的职责。

交互规则：

- 地图使用 `location.png` 作为背景底图。
- 每个建筑在地图上显示为书法标签和定位点。
- 鼠标悬停、按下或聚焦建筑标签时，显示该建筑的高亮预览。
- 底部浮层同步显示该建筑的导览文案摘要。
- 右上角目录抽屉提供另一种进入建筑的方式。

设计意图：

- 地图标签直接分布在建筑位置上，减少用户在“看地图”和“读目录”之间来回切换。
- 预览只在 hover 或 focus 时出现，避免总览态信息过载。
- 地图高亮不是简单描边，而是裁切建筑区域并上浮，强化“循图入景”的感觉。

### 4.2 单体态

单体态用于承载单个建筑的深入浏览。

交互规则：

- 中央区域加载并展示当前建筑 GLB 模型。
- 模型加载完成后，先播放一段自动旋转的开场动画，再开放用户控制。
- 用户可拖拽旋转模型、滚轮缩放模型。
- 左上区域展示建筑标题与摘要。
- 解说面板默认展开，使用逐字打字机效果呈现文本。
- 若存在音频文件，解说面板会自动检查并提供音频开关。
- 目录抽屉在单体态仍可用，支持跨建筑切换。

设计意图：

- 自动旋转先帮助用户建立完整形体印象，再交还控制权。
- 文案和模型同时出现，但通过留白和纸本面板分层，避免互相抢占注意力。
- 音频作为增强能力而非强制播放逻辑，保留用户控制权。

### 4.3 墨迹转场

总览态与单体态之间，以及建筑与建筑之间，都通过墨迹覆盖再揭开的方式切换。

设计作用：

- 解决模型切换时的“技术性闪烁”，让切换更像章节推进。
- 将中国古典园林的纸墨气质延续到动效层。
- 用转场文案显示当前进入的场景名称，强化导览节奏。

## 5. 视觉设计

### 5.1 视觉关键词

- 纸本
- 水墨
- 留白
- 温润
- 文人气

### 5.2 颜色策略

当前视觉以浅绢纸色和棕墨色为主：

- 背景主色偏米杏与淡绢纸色。
- 文字主色偏深棕与墨色，保证阅读稳定性。
- 高亮和氛围层使用暖白、淡金、浅褐的低饱和渐变。

设计原则：

- 不使用高饱和对比色破坏古典气质。
- 使用多层渐变和模糊高光模拟纸面、光晕与雾气，不依赖单色底。

### 5.3 字体策略

- 标题与地图标签使用 `Ma Shan Zheng`，强化书写感和东方文化识别度。
- 正文与说明文字使用系统无衬线字体，保证可读性和加载稳健性。

### 5.4 组件风格

- 面板与按钮统一采用“纸片卡 + 轻边框 + 柔和投影”样式。
- 圆角偏大，强调温和、卷轴式、纸本式体验。
- 交互反馈避免硬朗弹跳，更倾向慢速、缓出的位移和透明度变化。

## 6. 内容设计

### 6.1 文案结构

每个建筑目前包含三层内容：

- 概览层：用于总览态悬停预览。
- 摘要层：用于单体态标题卡片。
- 解说层：用于单体态深入阅读和配音。

### 6.2 内容风格

- 使用较强的文学表达与园林术语。
- 同时兼顾建筑形制、造园技法和文化意境三个维度。
- 通过标题式分段提升长文本可扫读性。

## 7. 响应式与可用性

### 7.1 响应式原则

- 地图始终保持整图铺满视窗，优先保证空间结构完整。
- 目录与说明面板使用 `min()`、`max()` 和视口约束，适配不同屏幕尺寸。
- 移动端将更多交互集中在边缘与抽屉，减少对中心模型区的遮挡。

### 7.2 可访问性考虑

- 所有可点击建筑标签都提供 `aria-label`。
- 目录按钮和解说开关支持键盘聚焦。
- 模型加载、完成、失败都有明确状态文案反馈。

## 8. 当前版本范围

### 8.1 已实现

- 园林总览地图入口
- 建筑地图标签定位
- 建筑目录抽屉
- 建筑预览卡片
- GLB 模型动态加载与展示
- 模型加载进度与异常反馈
- 模型自动开场旋转
- 建筑解说打字机效果
- 可选音频旁白
- 墨迹转场

### 8.2 暂未实现

- 建筑搜索与筛选
- 多园林或多主题切换
- 模型热点标注
- 游览历史、收藏、分享
- 多语言内容
- 后台内容管理

## 9. 后续设计建议

### 9.1 体验增强

- 为单体模型增加热点讲解点，支持按部位或路径导览。
- 增加“推荐游线”模式，例如春景线、借景线、文人书斋线。
- 给总览态增加昼夜、四季或节气主题皮肤，提升复访价值。

### 9.2 内容增强

- 为每个建筑补充更多历史图片、诗文出处和空间关系说明。
- 增加整体园林导览音频，而不是只在建筑级别提供音频。

### 9.3 运营与展示增强

- 补充分享海报或二维码模式，适配展馆与活动现场传播。
- 为大屏展示增加自动轮播导览模式。
